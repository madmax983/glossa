# Echo's DX Audit Report 🗣️

## Summary
The "Quick Start" experience is broken. The narrative "Hero's Journey" fails at Chapter 4 due to undefined variables. The "Troubleshooting" guide promises friendly Greek error messages, but the compiler often throws raw Rust errors or silently ignores issues. The testing framework's equality assertion (`ἰσοῦται`) appears to be a no-op, allowing tests to pass even when values differ.

## 1. The Broken Hero's Journey (Quick Start)

### Issue
Chapter 4 of the Quick Start guide fails to compile.

**Code:**
```glossa
// Chapter 4: The Logic (Control Flow)
// If age is greater than 50...
εἰ ἡλικία 50 μεῖζον ᾖ,
    «σοφός» λέγε.
```

**Result:**
`Rustc Error: mismatched types`. The variable `ἡλικία` is undefined. It was defined as a *field* of the `Χρήστης` struct in Chapter 3, but not as a standalone variable. The user is left stranded.

### Fix Suggestion
Update the example to correctly access the field from the instance created in Chapter 3 (e.g., `χρήστης ἡλικία` if supported, or define a variable `η` first), or adjust the narrative to define `ἡλικία` explicitly.

## 2. Troubleshooting Guide vs. Reality

The "Troubleshooting" table promises specific error messages, but the reality is different.

| Error Message | Documented Behavior | Actual Behavior |
|---------------|---------------------|-----------------|
| **Ἀσυμφωνία** | Subject/Verb mismatch | ✅ Works as expected. |
| **Διπλοῦν ὑποκείμενον** | Double Subject | ❌ **Silent Success.** `ὁ ἄνθρωπος ὁ θεός λέγει.` compiles and runs without error. |
| **Οὐκ οἶδα τὸ ὄνομα** | Undefined variable | ❌ **Rust Error / Silent.** `αγνωστος λέγε.` runs silently (prints nothing?). `εἰ ...` with undefined var causes `Rustc Error`. |
| **Ῥῆμα οὐχ εὑρέθη** | Verb not found | ❌ **Rust Error.** `ὁ ἄνθρωπος.` causes `Rustc Error: cannot find value ...`. |

### Fix Suggestion
Ensure semantic analysis catches these errors *before* codegen to provide the promised friendly messages. "Leaky abstractions" (raw Rust errors) are a major friction point.

## 3. The Testing Mirage

The testing framework looks beautiful but `ἰσοῦται` (equals) seems to be broken.

**Code:**
```glossa
δοκιμή «fail with vars».
    α 1 ἔστω.
    β 2 ἔστω.
    α β ἰσοῦται. // Should fail!
τέλος.
```

**Result:**
`✓ Πᾶσαι αἱ δοκιμασίαι ἐπέτυχαν! (All tests passed)`

The test PASSED despite 1 not equaling 2. This suggests `ἰσοῦται` is not asserting equality (e.g., `assert_eq!`) but perhaps just evaluating an expression that is ignored.

### Fix Suggestion
Ensure `ἰσοῦται` generates `assert_eq!` or equivalent panic-inducing code in test mode.

## Conclusion
The current DX is fragile. New users will hit immediate blockers following the README. The "Troubleshooting" section is misleading, and the testing tool gives a false sense of security.
