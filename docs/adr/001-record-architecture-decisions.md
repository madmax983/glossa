# 1. Record Architecture Decisions

Date: 2024-10-24
Status: Accepted

## Context

We need to record architectural decisions, the context of those decisions, and the consequences. This will help current and future contributors understand why the system is built the way it is.

## Decision

We will use Architecture Decision Records (ADRs) to track important architectural decisions.

We will use the format described by Michael Nygard:
- **Title**: Short noun phrase.
- **Context**: The forces at play, including technological, political, social, and project local.
- **Decision**: The response to these forces.
- **Status**: Proposed, Accepted, Deprecated, Superseded, etc.
- **Consequences**: The resulting context, after applying the decision.

## Consequences

- We will have a clear history of decisions.
- New contributors can learn the "why" behind the code.
- It requires discipline to write them.
