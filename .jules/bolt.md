## [Pre-allocated Statement Analysis]
**Learning:** In AST analysis phases, we frequently iterate over AST nodes like `block_stmts` to generate an array of analyzed models. Using `Vec::new()` requires the memory allocator to resize and copy the vector contents dynamically as new elements are added.
**Action:** When mapping or extending a known upper-bound size like block length `block_stmts.len()`, replacing `let mut analyzed = Vec::new();` with `Vec::with_capacity(block_stmts.len())` acts as a zero-cost abstraction, immediately preventing unnecessary array reallocations without any API disruption.
